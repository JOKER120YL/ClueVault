# ClueVault v2 正式 MVP 实施计划

## 1. 开发目标

把流程 Demo 中确认的“悬浮窗极速归档”落到 WPF 主线。

第一轮验收只看一件事：运营能否在 Windows 桌面上把微信群收到的模型文件快速归档到共享盘对应专业目录。

## 2. 阶段划分

### Phase 1：归档核心服务

目标：先把文件归档规则做稳。

任务：

- 将现有 `BugSubmissionService` 收敛为面向归档的 `ArchiveService`。
- 字段从“问题单”改为“用户模型归档”。
- 专业是唯一必选业务字段。
- 来源默认 `微信群`。
- 项目名、备注可空。
- 输出文件改为 `归档信息.md`。
- 归档目录按专业自动创建。
- 共享盘不可写时抛出明确错误。
- 失败不清空待归档文件。

验收：

- 能复制一个或多个文件到 `共享盘根目录/专业/归档目录/`。
- 能生成 `归档信息.md`。
- 共享盘不可写时不会丢失待归档文件。

### Phase 2：独立快速归档面板

目标：拖入文件后不打开主窗口，只弹出小面板。

任务：

- 将现有 `QuickSubmitWindow` 改为 `QuickArchiveWindow`。
- 面板只默认显示待归档文件和专业按钮。
- 专业使用紧凑按钮，不使用下拉。
- 项目名、来源、备注放进“更多信息”折叠区。
- 主按钮文案为 `归档到共享盘`。
- 归档成功后显示目录、文件数和打开目录入口。
- 归档失败后保留文件列表和专业选择。

验收：

- 从悬浮窗拖入文件后只出现快速归档小面板。
- 不自动打开主窗口。
- 三个关键动作能完成：拖入、选专业、归档。

### Phase 3：透明悬浮窗

目标：悬浮窗收起态只显示卡通角色。

任务：

- 移除当前 WPF 悬浮窗里的圆形背景、文字条和多余矩形容器。
- 保留 `AllowsTransparency=True`、`Background=Transparent`、`Topmost=True`。
- 收起态只渲染角色图片。
- 鼠标悬停不展开。
- 点击角色只展开状态或打开轻量状态，不打开主窗口归档流程。
- 拖入文件时触发快速归档小面板。
- 保留蛙弟 / 蜂哥自动切换逻辑。
- 预留 `DragOver` 状态，用于后续“张开大口”素材或动画。

验收：

- 悬浮窗背景透明。
- 收起态视觉上只有角色。
- 达到当日阈值后自动切换蜂哥。
- 拖入文件可以触发归档。

### Phase 4：今日归档与备注同步

目标：归档后能回看，并能补备注。

任务：

- 将历史记录字段改为归档语义。
- 今日记录显示时间、专业、文件数、目录、备注。
- 支持重新编辑备注。
- 保存备注后重写对应目录的 `归档信息.md`。
- 目录不存在时提示“原目录不可访问”，但保留本地记录。

验收：

- 今日归档记录能看到刚归档的文件。
- 修改备注后，目标目录里的 `归档信息.md` 同步变化。

### Phase 5：截图辅助开关

目标：保留截图辅助，但不影响极速归档。

任务：

- 配置中心增加 `拖入后截图` 开关。
- 开启后，拖入文件后先提示截图。
- 第一轮可以先实现为手动添加截图文件或占位流程。
- 真正框选屏幕截图可以作为 Phase 5 的后半段实现。

验收：

- 关闭截图时，归档路径不多一步。
- 开启截图时，截图会作为附件进入归档目录。

### Phase 6：经验库独立入口

目标：经验库不干扰模型归档。

任务：

- 主窗口保留经验库菜单。
- 支持手动记录“用户怎么问、怎么回、适用场景”。
- 支持本地导入 / 导出 JSON。
- 大模型 API 和截图总结暂时只保留配置入口或占位说明，不进入归档主流程。

验收：

- 经验库能独立打开、保存、导入、导出。
- 提交模型时不会要求填写经验内容。

## 3. 优先级

P0：

- 归档核心服务。
- 快速归档面板。
- 透明悬浮窗。
- 今日归档记录。

P1：

- 备注同步。
- 截图辅助开关。
- 打开归档目录。

P2：

- 框选截图。
- 经验库导入 / 导出。
- 大模型 API 辅助总结。

## 4. 第一轮代码改造清单

建议按以下顺序改：

1. `Infrastructure/AppModels.cs`：改造模型命名，把 `Submission` 语义逐步替换为 `Archive` 语义。
2. `Infrastructure/AppServices.cs`：新增或重命名归档服务，输出 `归档信息.md`。
3. `QuickArchiveWindow.xaml` 与 `.cs`：紧凑快速归档面板。
4. `FloatingWidgetWindow.xaml` 与 `.cs`：改为透明角色入口。
5. `App.xaml.cs`：拖入文件后打开快速归档面板，不再强制唤起主窗口。
6. `MainWindow.xaml` 与 `.cs`：保留配置、今日归档、经验库，弱化问题单工作台。

## 5. 不变量

开发过程中不能破坏以下原则：

- 未选择专业不能归档。
- 未拖入文件不能归档。
- 共享盘不可写不能假装成功。
- 归档失败不能清空待归档文件。
- 主窗口不能成为日常归档必经路径。
- 经验库不能混进模型归档主流程。

## 6. 开发模型建议

- 架构改动、WPF 窗口行为、共享盘写入：使用强模型。
- 重命名、XAML 微调、文案调整、简单 bug：可以切到便宜模型。
- 每完成一个 Phase，用强模型做一次 review。
